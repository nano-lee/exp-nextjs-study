Linking과 Navigating

Next.js 라우터를 사용하면 페이지 간에 SPA와 유사한 클라이언트 사이드 라우팅을 할 수 있습니다.

클라이언트 사이드 라우팅을 위한 Link 컴포넌트가 제공됩니다.

import Link from "next/link";

function Home() {
    return (
        <ul>
            <li>
                <Link href="/">Home</Link>
            </li>
            <li>
                <Link href="/about">About Us</Link>
            </li>
            <li>
                <Link href="/blog/hello-world">Blog Post</Link>
            </li>
        </ul>
    );
}

export default Home;

위의 예시에서는 여러 개의 링크를 사용합니다. 각 링크는 알고있는 페이지에 대한 경로(href)를 매핑합니다.

  • / → pages/index.js
  • /about → pages/about.js
  • /blog/hello-world → pages/blog/[slug].js

뷰포트에 있는 모든 <Link />는 기본적으로 Static Generation을 사용하는 페이지를 미리 불러옵니다.(해당 데이터도 포함) 서버사이드 렌더링 라우트는 <Link />를 클릭한 경우에만 불러옵니다.

동적 경로에 연결하기​

인터폴레이션을 통해 경로를 생성할 수도 있는데, 이는 동적 라우팅 세그먼트에 유용합니다.

import Link from "next/link";

function Posts({ posts }) {
    return (
        <ul>
            {posts.map((post) => (
                <li key={post.id}>
                    <Link href={`/blog/${encodeURIComponent(post.slug)}`}>
                        {post.title}
                    </Link>
                </li>
            ))}
        </ul>
    );
}

export default Posts;

이 예시에서는 경로가 utf-8과 호환되도록 하기 위해 encodeURIComponent를 사용합니다.

대신, 다음과 같이 href에 URL 객체를 사용할 수도 있습니다.

import Link from "next/link";

function Posts({ posts }) {
    return (
        <ul>
            {posts.map((post) => (
                <li key={post.id}>
                    <Link
                        href={{
                            pathname: "/blog/[slug]",
                            query: { slug: post.slug },
                        }}
                    >
                        {post.title}
                    </Link>
                </li>
            ))}
        </ul>
    );
}

export default Posts;
  • pathname: pages 디렉토리에 잇는 페이지 경로명입니다. 이 경우는 /blog/[slug]입니다.
  • 'query': 동적 세그먼트를 포함하는 객체입니다. 이 경우는 slug입니다.

라우터 주입하기​

예제
  • Dynamic Routing

리액트 컴포넌트에서 'router' 객체에 접근하려면 useRouter 또는 withRouter를 사용할 수 있습니다.

일반적으로 useRouter를 사용하는 것을 권장합니다.

명령형(Imperative) 라우팅​

예제
  • Using Router

next/link로 대부분의 라우팅 요구사항을 충족할 수 있지만, 이것 없이도 클라이언트 사이드 네비게이션이 가능합니다.

다음 에시는 useRouter 훅을 사용하여 페이지를 이동하는 기본적인 방법을 보여줍니다.

import { useRouter } from "next/router";

export default function ReadMore() {
    const router = useRouter();

    return (
        <button onClick={() => router.push("/about")}>
            Click here to read more
        </button>
    );
}

얕은 라우팅​

예제
  • Shallow Routing

얕은 라우팅을 통해 getServerSideProps, getStaticProps, getInitialProps와 같은 데이터 페칭 메소드를 다시 실행하지 않고 URL을 변경할 수 있습니다.

useRouter 또는 withRouter를 통해 가져온 router 객체로부터 다른 state의 손실 없이 업데이트된 pathname과 query를 받아올 수 있습니다.

얕은 라우팅을 사용하기 위해서는 shallow 옵션을 true로 설정해야 합니다. 다음 예시를 살펴봅시다.

import { useEffect } from "react";
import { useRouter } from "next/router";

// 현재 URL은 '/'
function Page() {
    const router = useRouter();

    useEffect(() => {
        // 첫번째 렌더 이후에 항상 네비게이션을 수행합니다.
        router.push("/?counter=10", undefined, { shallow: true });
    }, []);

    useEffect(() => {
        // counter가 변경되었습니다!
    }, [router.query.counter]);
}

export default Page;

URL은 /?counter=10으로 업데이트되지만, 페이지는 교체되지 않습니다. 오직 라우터의 상태만 변경됩니다.

아래처럼 componentDidUpdate를 사용하여 URL 변경을 확인할 수도 있습니다.

componentDidUpdate(prevProps) {
  const { pathname, query } = this.props.router
  // 무한루프를 막기 위해 props가 변경되었는지 확인합니다.
  if (query.counter !== prevProps.router.query.counter) {
    // 새로운 query를 사용하여 데이터를 가져옵니다.
  }
}

주의​

얕은 라우팅은 현재 페이지의 URL 변경에 대해서만 동작합니다. 예를 들어 pages/about.js라는 다른 페이지가 있다고 가정해봅시다. 그리고 다음과 같이 실행한다면:

router.push("/?counter=10", "/about?counter=10", { shallow: true });

새로운 페이지로 라우팅했기 때문에 얕은 라우팅을 요청했음에도 불구하고 현재 페이지 대신 새로운 페이지를 로드하고 데이터를 새로 불러옵니다.

얕은 라우팅을 미들웨어와 함께 사용하면 새로운 페이지가 기존 페이지와 일치하는지 확인할 수 없습니다. 이는 미들웨어가 페이지를 동적으로 재작성할 수 있기 때문에 얕은 라우팅으로 건너뛴 데이터 페칭 없이는 이를 클라이언트 사이드에서 확인할 수 없기 때문입니다. 따라서 얕은 라우팅은 항상 얕은 라우팅으로만 처리되어야 합니다.
