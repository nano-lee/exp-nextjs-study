Form을 사용하면 웹 애플리케이션에서 데이터를 생성하고 업데이트할 수 있습니다. Next.js는 API Route를 사용하여 양식 제출 및 데이터 변형을 처리하는 강력한 방법을 제공합니다.

알아두면 좋은 정보:

  • 앱 라우터에서 양식 제출 및 데이터 변형을 처리하는 데 서버 액션을 사용할 것을 점진적으로 권장할 예정입니다. 서버 액션을 사용하면 API Route를 수동으로 만들 필요 없이 컴포넌트에서 직접 호출할 수 있는 비동기 서버 함수를 정의할 수 있습니다. = API Route는 CORS header를 지정하지 않으므로 기본적으로 동일 출처만 지정합니다.
  • API Route는 서버에서 실행되므로 클라이언트에 노출하지 않고 환경 변수를 통해 API 키와 같은 민감한 값을 사용할 수 있습니다. 이는 애플리케이션의 보안에 매우 중요합니다.
app-router action

Actions​

Actions는 리액트의 실험적인 기능입니다, 사용자 상호작용의 응답으로 비동기코드를 실행할 수 있도록 해줍니다.

Actions는 Next.js나 리액트 서버 컴포넌트의 특별점이 아니지만, 아직 안정적인 리액트 버전에서는 사용하지 못합니다. Actions를 Next.js를 통해 사용할 때, 리액트의 experimental 채널을 사용하게 됩니다.

Actions는 요소 내부의 action 인자를 통해 정의됩니다. 일반적으로 HTML form을 작성할 때, URL을 action 인자로 넣어줄 수 있습니다. Actions를 통해, 리액트는 함수를 즉시 넣어줄 수 있도록 해줍니다.

리액트는 Actions와 함께 최적화된 업데이트를 위해 사용 가능한 내장 함수들을 제공하고 있습니다. 아직 새로운 패턴이 개발 중에 있고 새로운 API가 추가될 수 있음을 알고 있어야 합니다.

Form Actions​

Actions는 웹 표준 <form>API에 맞게 확장되었고, 즉시 실행되는 점진적 향상과 로딩 상태를 가능하게 해줍니다. 이는 HTML 원시 요소인 formaction와 유사하게 동작합니다.

Server Functions​

서버 단에서 동작하며, 클라이언트 단에서 호출될 수 없는 함수입니다.

Server Actions​

Server Functions은 action으로 불립니다.

Server Actions는 form 요소의action 속성을 통해 점진적으로 향상될 수 있습니다. 이 form은 클라이언트 단의 Javascript가 로딩되기 전에 작동합니다. 이는 이 form의 제출에 리액트의 수화가 필요하지 않음을 뜻합니다.

Server Mutations​

데이터를 변경하고 redirect, revalidatePath, 혹은 revalidateTag를 호출하는 Server Actions을 말합니다.

예시​

Server-only Forms​

페이지 라우터를 사용하면 서버에서 안전하게 변경되는 데이터를 처리하기 위해 API 엔드포인트를 수동으로 생성해야 합니다.

import type { NextApiRequest, NextApiResponse } from 'next'

export default async function handler(
  req: NextApiRequest,
  res: NextApiResponse
) {
  const data = req.body
  const id = await createItem(data)
  res.status(200).json({ id })
}

그런 다음 이벤트 핸들러를 사용하여 클라이언트에서 API Route를 호출합니다:

import { FormEvent } from 'react'

export default function Page() {
  async function onSubmit(event: FormEvent<HTMLFormElement>) {
    event.preventDefault()

    const formData = new FormData(event.currentTarget)
    const response = await fetch('/api/submit', {
      method: 'POST',
      body: formData,
    })

    // 필요하면 response 핸들링
    const data = await response.json()
    // ...
  }

  return (
    <form onSubmit={onSubmit}>
      <input type="text" name="name" />
      <button type="submit">Submit</button>
    </form>
  )
}

Redirecting​

If you would like to redirect the user to a different route after a mutation, you can redirect to any absolute or relative URL:

Mutation 후 사용자를 다른 경로로 리디렉션하려는 경우 절대 또는 상대 URL로 리디렉션할 수 있습니다:

import type { NextApiRequest, NextApiResponse } from 'next'

export default async function handler(
  req: NextApiRequest,
  res: NextApiResponse
) {
  const id = await addPost()
  res.redirect(307, `/post/${id}`)
}

Form Validation​

기본 양식 유효성 검사에는 required 및 type="email"과 같은 HTML 유효성 검사를 사용하는 것이 좋습니다.

향상된 server-side validation의 경우 zod와 같은 스키마 validation 라이브러리를 사용하여 구문 분석된 양식 데이터의 구조를 유효성 검사합니다:

const schema = z.object({
  // ...

  // ...
}

Displaying Loading State​

서버에서 Form이 제출될 때 React state를 사용하여 로딩 상태를 표시할 수 있습니다:

import React, { useState, FormEvent } from 'react'

export default function Page() {
  const [isLoading, setIsLoading] = useState<boolean>(false)

  async function onSubmit(event: FormEvent<HTMLFormElement>) {
    event.preventDefault()
    setIsLoading(true) // 요청이 시작될 때 로딩을 true로 설정

    try {
      const formData = new FormData(event.currentTarget)
      const response = await fetch('/api/submit', {
        method: 'POST',
        body: formData,
      })

      // 필요한 경우 응답 처리
      const data = await response.json()
      // ...
    } catch (error) {
      // 필요한 경우 에러 처리
      console.error(error)
    } finally {
      setIsLoading(false) // 요청이 완료되면 로딩을 false로 설정합니다.
    }
  }

  return (
    <form onSubmit={onSubmit}>
      <input type="text" name="name" />
      <button type="submit" disabled={isLoading}>
        {isLoading ? 'Loading...' : 'Submit'}
      </button>
    </form>
  )
}

Error Handling​

React state를 사용하여 Form 제출이 실패할 때 오류 메시지를 표시할 수 있습니다:

import React, { useState, FormEvent } from 'react'

export default function Page() {
  const [isLoading, setIsLoading] = useState<boolean>(false)
  const [error, setError] = useState<string | null>(null)

  async function onSubmit(event: FormEvent<HTMLFormElement>) {
    event.preventDefault()
    setIsLoading(true)
    setError(null) // 새 요청이 시작될 때 이전 오류 지우기

    try {
      const formData = new FormData(event.currentTarget)
      const response = await fetch('/api/submit', {
        method: 'POST',
        body: formData,
      })

      if (!response.ok) {
        throw new Error('Failed to submit the data. Please try again.')
      }

      // 필요한 경우 응답 처리
      const data = await response.json()
      // ...
    } catch (error) {
      // 사용자에게 표시할 오류 메시지를 캡처하기
      setError(error.message)
      console.error(error)
    } finally {
      setIsLoading(false)
    }
  }

  return (
    <div>
      {error && <div style={{ color: 'red' }}>{error}</div>}
      <form onSubmit={onSubmit}>
        <input type="text" name="name" />
        <button type="submit" disabled={isLoading}>
          {isLoading ? 'Loading...' : 'Submit'}
        </button>
      </form>
    </div>
  )
}

Setting Cookies​

response의 setHeader 메서드를 사용하여 API Route 내에서 쿠키를 설정할 수 있습니다:

import type { NextApiRequest, NextApiResponse } from 'next'

export default async function handler(
  req: NextApiRequest,
  res: NextApiResponse
) {
  res.setHeader('Set-Cookie', 'username=lee; Path=/; HttpOnly')
  res.status(200).send('Cookie has been set.')
}
export default async function handler(req, res) {
  res.setHeader('Set-Cookie', 'username=lee; Path=/; HttpOnly')
  // Access-Control-Expose-Headers (CORS응답 헤더에 포함되지 않은 정보를 클라이언트가 엑세스 할 수 있게 하기 위한 옵션)
  res.status(200).send('Cookie has been set.')
}

Deleting Cookies​

esponse의 setHeader 메서드를 사용하여 API Route 내에서 쿠키를 삭제할 수 있습니다:

import type { NextApiRequest, NextApiResponse } from 'next'

export default async function handler(
  req: NextApiRequest,
  res: NextApiResponse
) {
  res.setHeader('Set-Cookie', 'username=; Path=/; HttpOnly; Max-Age=0')
  res.status(200).send('Cookie has been deleted.')
}
